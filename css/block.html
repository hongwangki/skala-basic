div(display: block) vs div(display: inline) 비교

왼쪽은 기본 블록, 오른쪽은 div를 CSS로 인라인처럼 바꾼 예제. 아래에 inline-block도 같이 보여줌.

1) 블록(block)

요소가 한 줄을 차지하고, 다음 요소는 자동으로 아래로 내려감.

BLOCK div 1
BLOCK div 2
BLOCK div 3

특징: 기본적으로 가로폭이 부모를 꽉 채움(100% 느낌), 줄바꿈이 발생.

2) 인라인처럼(inline)

div지만 display: inline을 주면 한 줄에 붙어서 배치됨.

INLINE A
INLINE B
INLINE C
(참고) span도 inline

특징: 내용만큼만 폭을 차지하고 줄바꿈이 없음.
또한 width/height, 위아래 margin은 기대처럼 안 먹는 경우가 많음.

3) 보너스: inline-block

“한 줄에 붙이되, width/height/margin도 박스처럼 먹게” 하고 싶으면 inline-block.

INLINE-BLOCK 1
INLINE-BLOCK 2
INLINE-BLOCK 3
INLINE-BLOCK 4
